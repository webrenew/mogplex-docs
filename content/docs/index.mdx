Mogplex has two surfaces that work together:

- the **web app** is the shared control plane for repos, agents, routing,
  credentials, and observability
- the **CLI** is the local runtime for repo-in-place work from your terminal

You do not need to learn the whole product at once. The important first
decision is where the work starts.

## Start in the right surface

- Start in **Web App** when you need to connect GitHub, import repositories,
  author agents, configure triggers or automations, or inspect runs across your
  team.
- Start in **CLI** when you already have a repo locally and want Mogplex to
  review, edit, summarize, or operate on it from the terminal.
- Use **both** when you want hosted account setup and shared configuration from
  the web app, but day-to-day execution in the CLI.

## First-day path

If you want the shortest path to a working Mogplex setup, use this order:

1. Open the [web app](/web) and connect GitHub in [Settings](/web/settings).
2. Verify GitHub App coverage in [Installations](/web/installations), then sync
   repos in [Projects](/web/spaces).
3. Define or inspect your reusable agents in [Agents](/web/agents).
4. Choose a routing surface:
   - [Triggers](/web/triggers) for `one event -> one agent`
   - [Assignments](/web/assignments) for repo-bound standing work
   - [Automations](/web/automations) for multi-step workflows
5. Install the [CLI](/cli) and run a first local task in the repo.
6. Use [Observability](/web/observability) once work is live.

## Fastest possible first run

If you only want to prove the product is working end to end, install the CLI
and start a session. If you are not authenticated yet, `mogplex` walks you into
browser sign-in or local API-key setup.

```bash
curl -fsSL https://www.mogplex.com/install.sh | sh
mogplex
```

Success looks like this:

- `mogplex` opens an interactive session
- you can authenticate with either your Mogplex account or a provider key
- `mogplex exec "summarize this repository"` returns a normal one-off result

If you prefer shell-managed credentials, direct provider env vars still work:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
# or
export OPENAI_API_KEY=sk-...
mogplex
```
