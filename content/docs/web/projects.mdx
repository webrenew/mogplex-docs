Projects is the repo control plane for the web app.

It is where imported repositories show up, where additional workspaces can be
created, and where you launch a sandbox-backed workspace session.

## What you can do

- Sync repositories from GitHub into your default project
- Create additional projects and repo entries
- Detect monorepos and target sub-project roots
- Launch or reopen a sandbox-backed workspace
- Open a workspace chat pinned to a specific sandbox
- Configure per-repo settings, favorites, and visibility
- Attach agents and cron-style work to repos
- Open the sandbox management view to stop, inspect, or delete running records
