This page exists for older links that still point to the original Projects docs.

The current dashboard route is `/spaces`, and the active documentation lives at
[Projects](/web/spaces).

## What changed

Mogplex kept the product language around **Projects**, but the route itself now
lives under `/spaces`.

That means you will still see project-oriented wording in the product while the
URL path uses the newer route name.

## What lives on the current page

The `/spaces` surface is the operational hub for repo-backed work:

- import and sync repositories from GitHub
- group repos into named projects and workspaces
- search, filter, favorite, hide, and organize repos
- launch or stop preview sandboxes for a repo
- open a workspace session from the repo list
- manage repo settings such as root directory, dev command, env sync mode, and
  sandbox timeout
- create repo-scoped assignments and cron-style work from the dashboard

## When to use it

Start from [Projects](/web/spaces) when you are trying to answer one of these
questions:

- Which repositories are in Mogplex right now?
- Which project or workspace should this repo belong to?
- Is preview infrastructure linked and healthy for this repo?
- Where should I open a workspace session or assign recurring work?

This legacy docs route stays in place so older inbound links keep resolving to
the correct product area.
