---
title: Credential resolution
description: How mogplex resolves provider credentials and what `/logout` does.
---

On session start (`main.ts` → `createProviderFromConfig`):

1. **Env var first:** `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GOOGLE_GENERATIVE_AI_API_KEY`, `GROQ_API_KEY`, `MISTRAL_API_KEY`, `DEEPSEEK_API_KEY`, `XAI_API_KEY`, `COHERE_API_KEY`, `VERCEL_API_TOKEN`, etc. (see `packages/providers/src/factory.ts` `PROVIDER_ENV_KEYS` for the full map.)
2. **Fallback to `~/.mogplex/auth.json`** under `providers.<name>.apiKey` (set by `/login <provider>`).
3. **If both are absent**, the adapter fails at turn time with a clear `ANTHROPIC_API_KEY is required` error.

## Implication for `/logout`

- `/logout` only removes the `auth.json` entry.
- If the env var is set, the next session still authenticates via the env var. This is by design — env vars are the source of truth for shell-driven setups.
- Within the **current** session, the adapter is held in memory and reuses whatever key it was constructed with at startup. `/logout` does not force a reload by default. See `session.ts::reloadAdapter`.

## Grep anchors for audit

- `packages/providers/src/factory.ts::resolveApiKey` (env → auth.json)
- `packages/providers/src/factory.ts::PROVIDER_ENV_KEYS`
- `packages/cli/src/main.ts::main` (one-shot adapter creation)
- `packages/engine/src/session.ts::reloadAdapter`

## Session logging and credential safety

Session logs at `~/.mogplex/logs/<session-id>.jsonl` never contain:

- API keys (`ANTHROPIC_API_KEY`, etc.)
- OAuth tokens or PATs
- `auth.json` contents

All data flows through `@mogplex/auth/redact.ts` before serialization. Redaction applies to both the `data` payload and the `message` field.

When `MOGPLEX_DEBUG=1` is set, debug logs capture stream chunks and tool dispatch events. These may contain prompt content or tool output, so treat debug logs with the same care as your source code.
